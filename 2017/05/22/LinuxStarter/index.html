Linux快速入门

下面是我曾经困惑过，以及预计新手会遇到的一些知识点。

First of ALL

  1. 最简单的ls，cd等命令， 和Windows上的 dir，cd是几乎一样的。
  2. 在某个路径path下，你输入某些字符，按Tab键，会得到自动完成，匹配文件或者文件夹的名字。
  3. 常用技巧：sudo切换Super user权限。如果忘记加sudo，需要重复上一次的命令，用 sudo !!即可。
  4. 想知道之前执行过的命令，直接按键盘的Up和down键可以浏览上次执行的命令。或者你可以输入history查找命令的历史。
  5. 新手一般总是搞不清楚路径，说多少次都不清楚。路径path，就是文件(夹)的目录，在Windows上也有一样的概念，Directory，如果连文件路径就搞不清楚，那使用Linux系统将会有莫名的感伤，本文将会列出相关内容帮助理解此内容。
  6. 命令的执行需要根据你自己的环境进行变通，有些新手遇到麻烦问题，会去百度查找，有人告诉你，执行sudo rm -rf /，这确实能解决问题，但是你一定会一脸懵逼。
  7. 环境变量的生效问题，环境变量是系统启动时初始化完成的，他包含的配置文件有多处，优先级和作用范围不一样，生效的条件是重启，假定你表达式正确的情况下。通常建议新手直接设置/etc/profile中配置，这个是对应Windows的系统环境变量（感兴趣者请自行查询用户环境变量和系统环境变量等相关术语）。不建议花费时间纠结在哪个地方配置。那如果改完不重启，要立即生效怎么办？用source命令，也就是.(dot)命令 直接 $:. filename执行，他的前提是仅仅当前终端生效。
  8. sudo 命令需要输入密码，输入的时候，看似没有反应，也没有你期待的***符号出现，那是正常的现象，放心输入之后回车即可。不必大惊小怪，这是因为*nix系统设计就是如此，安全，连长度就不告诉你。

    File

文件的命令有创建文件，修改文件，删除文件，查看文件，搜索文件。

创建文件夹

mkdir name

创建文件

touch, gedit, vi, vim。注意区分，除第一个是普通命令只创建文件外，后面是文件编辑器，打开之后即可输入内容。

修改文件

自行搜索 vi 或者 nano 等文件编辑器的使用方法。

删除文件

rm 删除文件

rm -rf 删除整个目录，可以删除非空目录下所有的文件

一般正常的做法是对于重要文件，mv filename filenamebak，直接重命名备份。

查看文件

查看的命令有：

cat 命令下直接输出文件内容到终端

tac 你可以猜测，他输出什么

vi

more 可以向后翻页

less 可以前后翻页，可以查找文件，跟vi类似, 推荐使用

tail 可以直接查看文件的末尾N行

head 跟tail成对的命令

搜索文件

搜索文件名：

locate 此命令极快，前提是updatedb，生成了文件索引数据库缓存

ls | grep 正则表达式

find -name 文件名称正则表达式

搜索文件内容:

grep 正则表达式 | less

cat 文件名 | grep 正则表达式

vi 和 less 中可以使用 /和?，分别对应向前向后查找，非常实用。

使用vi进行查找

向下搜索

/ - 使用一个模式进行搜索，并定位到下一个匹配的文本

n - 向下查找下一个匹配的文本

N - 向上查找前一个匹配的文本

向上搜索

? - 使用模式进行搜索，并定位到前一个匹配的文本
n - 向上查找前一个匹配的文本

N - 向下查找下一个匹配的文本

简单记：n继续找，N上一个

Manage

进程管理

资源查看

top，可以实时监控进程的情况

w 查看资源负载

/proc/loadavg 上述命令本质上关联的还是此命令的输出，即平均负载。

例如：load average: 0.13, 0.18, 0.15

分别是 1分钟，5分钟，15分钟的负载，该数值越低，表明进程等待的时间越少，0.00时，立即可以获得CPU时间片。

进程管理

ps

  • -A 显示所有进程（等价于-e）
  • -a 显示一个终端的所有进程，除了会话的领导进程
  • -N 忽略选择。
  • -d 显示所有进程，但省略所有的会话的领导进程
  • -x 显示没有控制终端的进程，同时显示各个命令的具体路径。dx不可合用。
  • -p pid 进程使用cpu的时间
  • -u uid or username 选择有效的用户id或者是用户名
  • -g gid or groupname 显示组的所有进程。
  • U username 显示该用户下的所有进程，且显示各个命令的详细路径。如:ps U zhang;
  • -f 全部列出，通常和其他选项联用。如：ps -fa or ps -fx.
  • -l 长格式（有F,wchan,C 等字段）
  • -j 作业格式
  • -o 用户自定义格式。
  • v 以虚拟存储器格式显示
  • s 以信号格式显示
  • -m 显示所有的线程
  • -H 显示进程的层次(和其它的命令合用，如：ps -Ha)
  • e 命令之后显示环境（如：ps -d e; ps -a e）
  • h 不显示第一行
  • a 显示所有用户的所有进程
  • u 以用户为主的格式来显示程序状况
  • x 显示所有程序，不以终端机来区分
  • f 用ASCII字符显示树状结构，表达程序间的相互关系

为什么有的有-，有的没有呢？
有-的是standard syntax ，没有的是BSD syntax

常用

ps aux 显示所有用户的进程
ps afx 显示所有进程并且以树状结构显示
ps -ef 显示所有进程，显示有所列。

结束进程

  • kill kill -l列出所有signal 使用时直接-signal
  • pkill kill

说明：

  • -l 信号，若果不加信号的编号参数，则使用“-l”参数会列出全部的信号名称
  • -a 当处理当前进程时，不限制命令名和进程号的对应关系
  • -p 指定kill 命令只打印相关进程的进程号，而不发送任何信号
  • -s 指定发送信号
  • -u 指定用户

术语

路径

Windows的用户们，应该知道C:是盘符，C:\是C盘的路径。
Windows的目录是 Multi-Root 多个根目录，每一个逻辑磁盘都有一个根目录。
根目录就是顶层的目录，cd ..回不去的最高层目录。
而Linux的路径 只有一个根目录 也就是 /。

你必须知道的符号：

  1. ./表示当前的目录。
  2. ../表示上一级目录。
  3. .开头的文件一般是隐藏文件
  4. ls -al 显示文件的属性，9列信息，请自行查阅资料。

下面是重点：

目录就是记录文件逻辑位置的标记，就像地址一样。

你当前的终端(Linux的console,Windows的命令行)，也是路径相关的，Linux系统不知道你要访问哪里的文件，因此你必须知道你在哪个路径下。
不然你将面对无数的File not found ,Command not found. File not exsits. 这一类错误信息，只因为你的ignorance。

根路径不要忽略

如何知道在哪里呢？pwd。

那么有一些特殊的符号表示某些位置你需要了解：

  1. ~/表示/home/userName/，请联系Windows的我的文档文件夹, /home/userName/和user/username 少一个/，会出现cannot find path…。
  2. `pwd` , `uname -a` 类似这样的表达式，可以作为路径的变量。
  3. 环境变量里面 :表示路径分隔符，例如PATH=$PATH:/yourname/path/他的作用和Windows的环境变量分隔符;完全一样。

因此，如果掌握了路径这一概念，就能知道如何操作文件，访问命令不出错。

平均负载

平均负载就是系统单位时间内运行队列中就绪等待的进程数的平均值。

信号

只有第9种信号(SIGKILL)才可以无条件终止进程，其他信号进程都有权利忽略。

下面是常用的信号：
|—-|—–|——|
HUP 1 终端断线
INT 2 中断（同 Ctrl + C）
QUIT 3 退出（同 Ctrl + \）
TERM 15 终止
KILL 9 强制终止
CONT 18 继续（与STOP相反， fg/bg命令）
STOP 19 暂停（同 Ctrl + Z）
